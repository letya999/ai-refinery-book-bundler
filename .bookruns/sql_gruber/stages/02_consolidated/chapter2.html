Фильтрация и Сортировка

Точность данных — это не просто отсутствие ошибок, это умение отсекать лишнее. В SQL мастерство владения предложением WHERE определяет разницу между хаотичным дампом данных и хирургически точным отчетом.

Базовая фильтрация: WHERE

Предложение WHERE позволяет устанавливать предикаты — логические условия, которые должны быть истинными для включения строки в результат.

Синтаксис выбора: SELECT column1, column2 FROM table_name WHERE condition;

Сравнение инструментов: IN vs BETWEEN

Выбор правильного оператора повышает читаемость кода и производительность запроса.

Оператор IN

Идеален для поиска совпадений в дискретном наборе значений (списке).

WHERE city IN ('London', 'Barcelona', 'Rome')

Оператор BETWEEN

Определяет непрерывный диапазон (включая границы).

WHERE amt BETWEEN 1000 AND 5000

Логика NULL: Территория неизвестного

В SQL NULL — это не ноль и не пробел. Это отсутствие данных. Любое сравнение с NULL (кроме специальных операторов) дает результат UNKNOWN.

Value ↔ NULL comparison
WHERE col = NULL
❌ FALSE
➔
WHERE col IS NULL
✅ TRUE

Всегда используйте IS NULL или IS NOT NULL.

Маски и шаблоны: LIKE

Для поиска по неполным данным используются спецсимволы подстановки:

% — любая последовательность (0+ символов) _ — ровно один любой символ
Пример поиска по шаблону:

Найти всех заказчиков, чьи имена начинаются на 'C' и содержат 's':

SELECT * FROM Customers WHERE cname LIKE 'C%s%';

Краткий справочник операторов

Оператор Назначение Пример
=, <>, >, < Базовое сравнение rating >= 200
AND, OR, NOT Логические связки city='London' AND rating>100
IN Проверка по списку snum IN (1001, 1002, 1007)
BETWEEN Диапазон значений comm BETWEEN 0.10 AND 0.15
LIKE Поиск по шаблону sname LIKE 'P_e%'
IS NULL Проверка на пустоту amt IS NULL

Сортировка: ORDER BY

Упорядочивание вывода — финальный штрих профессионального отчета.

SELECT * FROM Orders ORDER BY amt DESC, odate ASC;

DESC — по убыванию, ASC — по возрастанию (по умолчанию).
