Модуль 4: Связи и JOINs

Реляционные БД хранят данные в разных таблицах. JOIN — это механизм их временного сшивания по ключу.

INNER JOIN

Только пересечение

LEFT JOIN

Все левые + совпадения

SELECT c.cname, o.amt
FROM Customers c
INNER JOIN Orders o ON c.cnum = o.cnum;

Матрица сложности и типов

Self-Join

Таблица связывается сама с собой через алиасы. Полезно для иерархий.

Сложность: Low

Cross Join

Декартово произведение (каждый с каждым). Редко нужно, вешает БД на больших данных.

Сложность: High (Load)

Equi-Join

Связь по строгому равенству =. Самый частый тип связи.

Сложность: Normal

Non-Equi Join

Связь по диапазонам BETWEEN или < / >. Поиск в интервалах.

Сложность: Normal

Золотые правила

  • Всегда проверяйте ON-условие. Забытый ключ превращает JOIN в CROSS JOIN.
  • Используйте краткие алиасы (c для Customers, o для Orders).
  • LEFT JOIN возвращает NULL для правой таблицы, если совпадений нет.
