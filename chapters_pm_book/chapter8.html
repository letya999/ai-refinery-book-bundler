WBS: Иерархическая структура работ

ИСР — это фундамент планирования. Если работы нет в ИСР, значит, она не входит в проект.
ПРИНЦИП Ориентация на результат

Как строить ИСР (WBS)

Разбивайте проект на управляемые пакеты работ. Пакет должен быть достаточно мелким для оценки, но не избыточным.

Метод набегающей волны

Детализируем ближайшие задачи, а далекие оставляем укрупненными блоками.

Словарь ИСР

Подробное описание каждого элемента структуры: что именно нужно сделать.

ОШИБКИ
  • Декомпозиция ради декомпозиции (слишком мелко)
  • Функциональный подход («программирование», «тестирование»)
  • Создание ИСР в одиночку без команды
ПРАВИЛА
  • Поставко-ориентированный подход («макет», «модуль Х»)
  • Совместное творчество с исполнителями
  • Разумная глубина (пакет работ)
Признаки хорошего пакета работ
  • Относительно короткий по времени.
  • Может быть выполнен без прерываний на внешние согласования.
  • Имеет четкий, измеримый результат.
