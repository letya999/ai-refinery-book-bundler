# Performance Playbook (Interactive Book)

## Обязательные практики

1. Анимации: использовать `requestAnimationFrame`, а не `setInterval` для рендера кадров.
2. Снижать стоимость отрисовки: не пересчитывать геометрию в каждом кадре, кэшировать контрольные точки/узлы.
3. Учитывать доступность: поддерживать `@media (prefers-reduced-motion: reduce)`.
4. Скрытые/дальние секции: применять `content-visibility` и разумные размеры контейнеров.
5. Длинные страницы: не вешать тяжелые scroll-listeners; по возможности использовать `IntersectionObserver`.
6. Любой пере-рендер графика должен очищать старое состояние (instance/timer/listeners).

## Для canvas-графики

- Разделять «статический слой» (фон/сетка/подписи) и «динамический слой» (подсветка/пульс).
- Ограничивать толщины линий и тени: они резко растят стоимость пиксельной отрисовки.
- Для тяжелых кейсов рассматривать `OffscreenCanvas` + worker (если поддержка браузера целевая).

## Для интерактива

- Каждому контролу (кнопка фазы/ползунок) давать моментальную визуальную обратную связь.
- При изменении состояния обновлять только затронутый блок.
- Не допускать «мерцаний» layout: заранее резервировать место под графики/панели.

## Проверка перед релизом

- Нет console errors.
- Нет заметных фризов при 60Hz прокрутке/кликах.
- Мобильный и десктопный рендер одинаково читаем.
- Движение можно уменьшить (reduced motion).
