# Library Injection Mode (When Needed)

Этот режим включать только если нативный `canvas + JS` заметно ухудшает поддержку/скорость разработки.

## Когда библиотека допустима

- сложная анимационная оркестрация (временные шкалы, синхронизация сцен);
- графики с большим количеством типов и легенд;
- повторяемый UI-эффект, который в vanilla будет громоздким.

## Принцип «минимальной инъекции»

1. Разрешены только заранее одобренные библиотеки.
2. Только фиксированные версии.
3. Только локальная инъекция в single-file (не runtime CDN).
4. Ленивая загрузка в момент входа в нужную главу.
5. Должен быть fallback без библиотеки (или деградация до статического вида).

## Рекомендации по выбору

- Анимации: сначала Web Animations API / CSS / canvas; библиотека только при реальной нехватке.
- Графики: нативный canvas для простых диаграмм, Chart.js — только при сложных комбинированных графиках.

## Lifecycle

- Создание: только при первом входе в главу.
- Обновление: переиспользовать instance.
- Удаление: при выходе освобождать instance/listeners/timers.

## Контроль размера

- Каждый injected модуль должен иметь обоснование «почему нельзя проще».
- Если библиотека используется <2-3 фичами, вероятно она лишняя.
