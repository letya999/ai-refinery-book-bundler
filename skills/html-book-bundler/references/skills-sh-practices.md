# skills.sh practices (applied to this skill)

Короткая памятка, какие практики учтены при развитии `html-book-bundler`.

## Что взято из skills.sh docs

- Навык должен быть практичным и процедурным: четкие шаги, а не абстрактные советы.
- Навык нужно валидировать фактическим прогоном (в нашем случае: browser QA + console check).
- Любой внешний скилл — не «истина по умолчанию», перед применением нужна проверка содержимого.

## Что взято из web-design-guidelines (skills.sh)

- Перед сдачей UI должен пройти системный review: визуальная ясность, доступность, поведение интерактива.
- Формат отчета о проблемах должен быть точечным и исправляемым (`файл:проблема:действие`).

## Как применяем в интерактивных книгах

1. Проверяем итоговый shell и все главы отдельно.
2. Фиксируем и исправляем консольные ошибки до релиза.
3. Делаем визуал «смысловым»: схема + подписи + короткое объяснение, а не декоративный рисунок.
4. Для preview сохраняем управленческую логику глав (выжимка + практическая методика + риски).
