# PDF Visual Workflow

Этот workflow нужен, когда книгу собирают не только из текста, но и из схем/рисунков.

## Шаг 1. Извлеките визуалы из PDF

```bash
python skills/html-book-bundler/scripts/extract_pdf_visuals.py \
  --pdf ./book.pdf \
  --out ./_pdf_assets \
  --pages 18-30 \
  --dpi 180
```

Результат:
- `pages/page_XXX.png` — скрин каждой страницы (для ориентира)
- `images/pXXX_iYY.*` — встроенные изображения
- `manifest.json` — метаданные

## Шаг 2. Определите, что переносить в интерактив

Для каждой схемы зафиксируйте:
1. Что это объясняет (смысл).
2. Какие сущности обязательны (узлы, стрелки, подписи).
3. Что интерактивно (hover, click, переключатель фазы, фильтры).

## Шаг 3. Выберите формат переноса

- **SVG** — для диаграмм и схем (рекомендуется).
- **Canvas** — для динамических/анимированных графиков.
- **PNG/JPG + оверлей** — если схему нельзя быстро воссоздать в векторе.

## Шаг 4. Встройте в главу автономно

Правило: никакие внешние URL. Если это растровый ассет, используйте относительный путь до сборки и убедитесь, что после bundling он вшит в B64-контент главы.

## Шаг 5. Проверка качества

- Смысл схемы понятен без чтения длинного текста.
- Подписи читаются на мобильном (минимум 12px).
- Цвет и контраст проходят базовую проверку доступности.
- Есть краткое текстовое объяснение под схемой.
